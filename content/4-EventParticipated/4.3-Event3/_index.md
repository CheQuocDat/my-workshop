# Event Report "FCAJ Community Day - July 11"

### Event Objective

The 3rd FCAJ Community Day was a highly valuable gathering for the AWS First Cloud AI Journey community. The event featured the dramatic grand finale of the Cloud Architect competition, along with in-depth technical sessions on web application security, operational optimization via SLA & Monitoring, and a roadmap to conquer the AWS Cloud Practitioner certification for future cloud careers.

### Event Information

- **Date**: Saturday, July 11, 2026
- **Time**: 09:00 AM - 12:00 PM
- **Location**: 26th Floor, Bitexco Financial Tower, Ho Chi Minh City
- **Role**: Attendee

---

### Detailed Schedule

#### 8:30 AM - 9:00 AM: Registration and Networking
Attendees checked in at the Bitexco Financial Tower and engaged in networking with fellow peers and industry speakers.

#### 09:00 AM - 09:40 AM: Cloud Architect Grand Finale (KLKAT vs Ngu Dai Hiep)
A highly anticipated showdown between the two top teams of the Cloud Architect program. The teams competed to solve real-world architectural design and optimization scenarios under judges' evaluation. Concurrently, an interactive AWS knowledge quiz was held for the audience in the hall.

#### 09:40 AM - 10:40 AM: Securing Your Web Apps With AWS Security Agent
**Speaker:** Thinh Nguyen (DevSecOps/Cloud Engineer)

Key Topics:
- Importance of web application security in modern cloud environments.
- Detailed architecture and core principles of the AWS Security Agent.
- Configuring security policies, continuous monitoring, and system vulnerability detection.
- DevSecOps best practices to integrate automated security checks into CI/CD pipelines.

#### 10:40 AM - 11:30 AM: SLA and Monitoring - From SLA to Monitoring what really matters
**Speaker:** Nguyen Huynh Son

Key Topics:
- Clarifying SLA (Service Level Agreement), SLO (Service Level Objective), and SLI (Service Level Indicator) in system operations.
- Constructing effective monitoring systems instead of simply collecting raw logs.
- Identifying critical technical metrics that reflect service quality and end-user experience.
- Designing intuitive monitoring dashboards and configuring automated alerts for incident management.

#### 11:30 AM - 12:00 PM: Inside The Exam: AWS Cloud Practitioner
**Speaker:** Ngo Le Tan Huy

Key Topics:
- Overview of the exam structure and key domains covered in the AWS Cloud Practitioner certification.
- Detailed learning roadmap, recommended study guides, and mock exam resources.
- Practical tips for taking the exam online or at testing centers.
- The professional value of AWS certifications for IT students entering the workforce.

---

### Key Learnings

#### Real-world Architecture Design Insights
Observing the Cloud Architect finale demonstrated how teams dissect business requirements, weigh the trade-offs of various cloud services, and propose architecture models balancing high availability, scalability, and cost optimization under time constraints.

#### Advancing Web Security Mindset (DevSecOps)
Mr. Thinh's session emphasized that security must be integrated continuously throughout the lifecycle, rather than being treated as an afterthought. Configuring security agents proactively detects and mitigates threats before they reach production.

#### Implementing Service-Oriented Monitoring
Mr. Son's presentation transformed my view on monitoring. A robust monitoring setup does not just alert when CPU spikes; it tracks user-facing metrics like HTTP 5xx error rates and API latency. This is essential for upholding SLA commitments.

#### Certification Strategies for AWS Cloud Practitioner
Mr. Huy's sharing provided a clear framework for my first AWS certification. I noted key domains to focus on, such as AWS pricing models, the Shared Responsibility Model, and core cloud services to confidently prepare for the exam.

### Personal Reflections

#### Active Learning and Competitive Spirit
The Cloud Architect finale brought a lively atmosphere with enthusiastic audience support. The interactive quiz was a fun, engaging way for everyone to challenge their AWS knowledge.

#### Direct Connection with Industry Professionals
I was inspired by the accessibility of the speakers. They generously stayed after their sessions to answer technical questions and share practical incident response stories from corporate environments.

### Takeaways & Action Items

- **Security Auditing for ITCoach**: Review the AWS WAF rules protecting the API Gateway and CloudFront distributions of our project to ensure robust data security.
- **Deploying Latency Monitoring**: Set up simple CloudWatch Metrics dashboards for ITCoach to track the response times of backend Lambda functions.
- **AWS Exam Preparation**: Register and start a structured self-study plan for the AWS Cloud Practitioner exam using the shared resources.
